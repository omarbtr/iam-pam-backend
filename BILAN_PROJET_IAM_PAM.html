3. Fonctionnalités critiques manquantes (CDC l'exige)
🔴
1. BASTION SSH/RDP — Absent entièrement (Section 4.1 + 4.4 + 5.2.2) C'est la fonctionnalité centrale du module PAM selon le cahier des charges. Actuellement, un utilisateur peut faire une demande approuvée, mais ne peut pas se connecter à la ressource. Il n'existe aucun mécanisme de proxy entre l'utilisateur et le serveur cible.
"Bastion Linux durci (proxy SSH/RDP)" — "Connexion au bastion avec identité utilisateur → Bastion établit connexion SSH/RDP vers cible" — "Injection automatique des credentials" — "Journalisation temps réel des commandes"
Ce qui manque concrètement :
  • Terminal web SSH dans l'interface (ex : xterm.js + WebSocket)
  • Proxy SSH côté backend (ex : Apache MINA SSHD ou Guacamole)
  • Container "bastion" dans le docker-compose
  • Enregistrement des commandes tapées en session
  • Table pam_sessions pour les sessions actives
🔴
2. Dockerfiles Backend & Frontend — Absents (Section 8.1) Le docker-compose.yml ne contient que les services d'infrastructure. Il n'existe pas de Dockerfile pour le Spring Boot ni pour l'Angular. Sans ces fichiers, il est impossible de conteneuriser l'application complète.
"Images Docker (Backend, Frontend, Keycloak, Bastion)"
🔴
3. Kubernetes — Absent entièrement (Section 4.1 + 8.1) Aucun manifeste Kubernetes dans le projet. Le CDC cite explicitement Kubernetes comme technologie d'orchestration et les manifestes comme livrable obligatoire.
"Manifestes Kubernetes (Deployments, Services, Ingress, ConfigMaps)" — "Déploiement conteneurisé (Docker/Kubernetes)"
🔴
4. Pipeline CI/CD — Absent (Section 8.1) Aucun fichier GitHub Actions, GitLab CI ou Jenkins dans le projet.
"Pipeline CI/CD" — listé comme livrable technique obligatoire
⚠️

7. Recommandation pour la soutenance
ℹ️
La soutenance est une présentation sans démo live. L'objectif est de montrer une couverture maximale du CDC dans le temps restant (Mois 4 → Juin 2026).
🗓️ Ordre de réalisation recommandé
1
Dockerfiles Backend + Frontend Multistage build : Maven → JRE pour le backend, Node build → Nginx pour le frontend. Intégrer dans le docker-compose. Effort : ~1 heure. Couvre un livrable obligatoire du CDC.
2
Dashboard Super-Admin Page Angular avec KPIs globaux (tenants actifs/inactifs, total utilisateurs, demandes en attente cross-tenant, dernières actions d'audit). Effort : ~2 heures.
3
Export CSV des logs d'audit Endpoint GET /api/auditor/logs/export + bouton Angular avec téléchargement. Effort : ~1 heure. Très visible pour les auditeurs de conformité.
4
Terminal SSH web (POC) Même un POC avec xterm.js + WebSocket qui ouvre une vraie connexion SSH via Apache MINA SSHD est suffisant pour la présentation. C'est le feature le plus démonstratif du PAM.
5
CI/CD GitHub Actions + Kubernetes (si temps) Un workflow simple (build → test → docker push) + 5-6 manifestes K8s de base. Couvre les livrables techniques du CDC même si non fonctionnel en demo.
